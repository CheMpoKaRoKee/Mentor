# Security Policy

## Общий принцип

`Learning Mentor` работает с локальными настройками, пользовательским прогрессом и внешним AI API. Поэтому главные риски связаны с конфигурацией, локальными данными и portable-сборками.

В репозитории должны храниться только исходный код, документация и безопасные примеры конфигурации.

## Что нельзя хранить в репозитории

Не добавляйте в git:

- `.env` с реальными значениями;
- приватные ключи доступа;
- пользовательские данные;
- папку `Portable/data`;
- runtime-логи;
- кэш Electron;
- готовые сборки из `dist/`, `release/`, `out/`;
- скриншоты с приватной информацией.

Для примеров конфигурации используйте только `.env.example`.

## Локальная конфигурация

Локальные настройки должны храниться вне публичных файлов.

`.gitignore` уже исключает:

- `.env`;
- `.env.*`;
- `dist/`;
- `release/`;
- `out/`;
- `Portable/*`, кроме служебного README;
- `data/`;
- `*.log`;
- Electron cache folders;
- `node_modules/`.

## Portable-сборка

Перед публикацией portable-архива проверьте:

- внутри нет локального env-файла;
- внутри нет пользовательского прогресса;
- внутри нет логов;
- внутри нет кэша Electron;
- внутри нет приватных настроек;
- архив проверен на тестовом запуске.

## Скриншоты и демо

Перед добавлением скриншотов проверьте, что они не содержат:

- личные настройки;
- приватные значения конфигурации;
- локальные пути с персональной информацией;
- пользовательские данные;
- историю запросов, которую не нужно публиковать.

Используйте тестовый профиль и нейтральные учебные темы.

## AI API

Рекомендации:

- не публикуйте значения доступа;
- не храните реальные значения в README;
- не добавляйте приватные настройки в portable-архив;
- проверяйте, что ошибки API не выводят лишнюю информацию в интерфейс или логи.

## Ответственное раскрытие

Если обнаружена проблема безопасности, исправьте её без публикации приватных данных. В публичном issue можно описывать только общий сценарий проблемы без чувствительных деталей.
