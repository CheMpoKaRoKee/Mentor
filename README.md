# Learning Mentor

Learning Mentor - настольное приложение для самостоятельного изучения fullstack-разработки. Программа помогает двигаться по учебному маршруту, получать теорию, выполнять проверочные задания и возвращаться к темам, которые требуют повторения.

Проект рассчитан на начинающих разработчиков, которым нужен компактный учебный инструмент без сложной настройки окружения. Приложение работает как desktop-программа на Electron и использует OpenRouter API для генерации уроков, проверки ответов и встроенного чата по текущей теме.

## Возможности

- пошаговый маршрут от HTML и CSS до React, Node.js, TypeScript, Docker и деплоя;
- генерация теории с примерами кода;
- проверочные задания в формате тестов и задач на код;
- автоматическое сохранение прогресса;
- повторение тем, которые не были пройдены с первого раза;
- встроенный чат-наставник во время изучения теории;
- portable-режим с хранением данных рядом с исполняемым файлом.

## Кому подойдет

- начинающим frontend- и fullstack-разработчикам;
- студентам, которым нужен персональный тренажер по базовым темам;
- тем, кто хочет регулярно закреплять материал небольшими уроками и экзаменами.

## Быстрый запуск из исходников

Требования:

- Node.js 20 или новее;
- pnpm;
- ключ OpenRouter API.

Установка зависимостей:

```bash
pnpm install
```

Создайте файл `.env` на основе `.env.example`:

```bash
cp .env.example .env
```

Заполните переменную:

```env
OPENROUTER_API_KEY=ваш_ключ_openrouter
```

Запуск в режиме разработки:

```bash
pnpm run dev
```

## Обычная установка

Если у вас есть готовый установщик из релиза:

1. скачайте `Learning Mentor Setup 1.0.0.exe` из раздела GitHub Releases;
2. запустите установщик;
3. откройте приложение;
4. получите OpenRouter API key на https://openrouter.ai/keys;
5. укажите ключ в настройках при первом запуске.

## Portable-версия

Portable-сборка находится в папке `Portable`. Она не требует установки: достаточно открыть `Learning Mentor.exe`.

Настройка OpenRouter API key в portable-версии:

- откройте https://openrouter.ai/keys;
- войдите в аккаунт OpenRouter и создайте новый ключ;
- вставьте ключ в окно настроек приложения;
- или создайте файл `.env` рядом с `Learning Mentor.exe` и добавьте строку `OPENROUTER_API_KEY=ваш_ключ_openrouter`;
- после изменения `.env` перезапустите приложение.

Особенности portable-режима:

- `portable.flag` включает хранение данных внутри папки `Portable\data`;
- прогресс и настройки создаются после первого запуска;
- папку можно перенести на другой диск или флешку;
- в git portable-сборка не добавляется из-за размера и должна публиковаться через GitHub Releases.

Для публикации portable-версии на GitHub загрузите архив папки `Portable` как asset релиза. В архиве должны быть `README-PORTABLE.txt` и `.env.example`, но не должно быть папки `data` с пользовательскими данными.

## Сборка

Сборка desktop-приложения:

```bash
pnpm run build
```

Результаты сборки создаются в `dist/`. Эта папка не хранится в репозитории.

## Настройки

Приложению нужен OpenRouter API key. Его можно указать двумя способами:

- через окно настроек внутри приложения;
- через переменную `OPENROUTER_API_KEY` в локальном `.env`.

Файл `.env` не должен попадать в репозиторий. Для примера настроек используется `.env.example`.

## Структура проекта

```text
learning-mentor/
├── src/
│   ├── main/          # главный процесс Electron, хранение данных, OpenRouter API
│   ├── preload/       # безопасный мост между Electron и React
│   └── renderer/      # интерфейс React, экраны и компоненты
├── Portable/          # локальная portable-сборка для релизов
├── electron-builder.yml
├── package.json
├── pnpm-lock.yaml
└── README.md
```

## Безопасность перед публикацией

Перед push проверьте:

- `.env` не добавлен в git;
- `Portable\data` отсутствует в portable-архиве;
- `node_modules`, `dist`, `release` и кэш Electron не попадают в коммит;
- ключ OpenRouter не сохранен в `settings.json`, `.env` внутри portable-папки или логах.
