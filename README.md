# Learning Mentor

**Learning Mentor** — desktop-приложение для самостоятельного изучения fullstack-разработки с AI-наставником. Программа помогает проходить учебный маршрут небольшими шагами: читать теорию, выполнять задания, проверять ответы, фиксировать прогресс и возвращаться к темам, которые требуют повторения.

## Кратко

Проект сделан как компактный учебный инструмент для начинающего разработчика. Он объединяет маршрут обучения, генерацию объяснений, проверочные задания и чат-наставник в одном desktop-интерфейсе.

Главная идея: не просто читать материалы, а регулярно проходить короткий цикл обучения:

1. выбрать тему;
2. получить объяснение и примеры;
3. выполнить проверочное задание;
4. сохранить прогресс;
5. повторить слабые темы позже.

## Case summary

**Проблема:** самостоятельное обучение часто распадается на разрозненные материалы, хаотичные AI-запросы и отсутствие проверки знаний.

**Решение:** Learning Mentor упаковывает AI-наставника в desktop-приложение с маршрутом, объяснениями, заданиями и сохранением прогресса.

**Результат:** пользователь получает не просто чат, а повторяемый учебный процесс: тема → объяснение → задание → вопрос наставнику → сохранение прогресса.

Подробнее: [Case Study](docs/case-study.md).

## Быстрый запуск из исходников

### Требования

- Node.js 20 или новее;
- pnpm;
- OpenRouter API key.

### Установка

```bash
pnpm install
```

Создайте `.env` на основе `.env.example`:

```bash
cp .env.example .env
```

Добавьте ключ OpenRouter:

```env
OPENROUTER_API_KEY=your_openrouter_api_key
```

Запуск в режиме разработки:

```bash
pnpm run dev
```

## Сборка приложения

```bash
pnpm run build
```

После сборки файлы создаются в `dist/`. Эта папка не хранится в репозитории.

## Portable-версия

Portable-сборка рассчитана на запуск без установки. Достаточно открыть исполняемый файл приложения.

Особенности portable-режима:

- `portable.flag` включает хранение данных внутри папки `Portable/data`;
- прогресс и настройки создаются после первого запуска;
- папку можно перенести на другой диск или флешку;
- пользовательские данные не должны попадать в git;
- готовую portable-сборку лучше публиковать через GitHub Releases как архив.

Перед публикацией portable-архива проверьте, что внутри нет:

- `.env` с реальным API-ключом;
- `Portable/data` с пользовательскими данными;
- логов, кэша и временных файлов;
- `node_modules`, `dist`, `release` и кэша Electron.

## Настройка OpenRouter

Приложению нужен OpenRouter API key. Его можно указать двумя способами:

1. через окно настроек внутри приложения;
2. через переменную `OPENROUTER_API_KEY` в локальном `.env`.

Файл `.env` не должен попадать в репозиторий. Для примера настроек используется `.env.example`.

## Безопасность перед публикацией

Перед `push` или публикацией релиза проверьте:

- `.env` не добавлен в git;
- OpenRouter API key не сохранен в `settings.json`, portable-папке или логах;
- `Portable/data` отсутствует в portable-архиве;
- `node_modules`, `dist`, `release` и кэш Electron не попадают в коммит;
- в README нет приватных данных, токенов и локальных путей пользователя.

Подробно: [Security Policy](SECURITY.md).
